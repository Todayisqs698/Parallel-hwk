并行向量检索优化实验 (Pthread & OpenMP 多线程)
本项目基于大容量近似最近邻搜索（Approximate Nearest Neighbor Search, ANNS）场景，在 DEEP100K 数据集 ($N=100,000$，$d=96$) 上，以 ARM NEON SIMD 优化为单线程算力底座，深入探究了 Pthread 静态/动态线程池 与 OpenMP 多线程 在不同索引架构下的并行设计、及加速比极限。
## 实验环境
* **硬件平台**
：x86 Windows 平台、鲲鹏架构 ARM64 服务器。
* **单线程基底**
：ARM NEON SIMD 向量化并行（4路展开、指令流水线优化）。
* **多线程库**
：POSIX Threads (Pthread)、OpenMP。
##  核心架构与并行优化方案

### 1. Flat-SIMD 与多线程结合
* **优化策略**：**Base 向量划分并行**
。采用 Pthread 创建常驻静态线程池，将底库向量通过静态步长交错循环均匀切分给各个线程处理。
* **数据流管理**：各工作线程独立维护大小为 $p$ 的私有局部最大堆；所有线程利用 `pthread_barrier`
 屏障同步收工后，由主线程进行串行规约（Reduce），合并全局 Top-K 结果。
* **加速比模型**
：
  $$\small S
_{flat}\approx\frac{T}{1+\frac{T\cdot(T\cdot p\cdot \log k\cdot t_{heap})}{(N\cdot d/f_{NEON})+(N\cdot \log p\cdot t_
{heap})}}$$
  由于存在串行规约的长尾占比，加速比随线程数呈现非线性碗型曲线。

### 2. PQ-SIMD 与多线程结合
* **优化策略**：**Batch-Query 任务划分并行**
。针对大批量查询场景，避免了单查询内部频繁同步带来的多阶段屏障开销。
* **无锁设计**
：维护固定大小的静态长效线程池，将批量查询以静态步长均匀分片。线程内部完全闭环执行单个查询的完整生命周期（构建 LUT -> 量化扫描 -> 维护局部粗排堆 -> 精排重排序）。由于不存在串行规约残余开销，具有极高的并行内聚性。

### 3. IVF 倒排索引多线程并行优化
基于 KMeans++ 聚类分桶建立倒排文件索引，将全局暴力扫描裁剪为两级检索（粗排选桶 + 精排扫桶）。
* **Pthread 静态并行**
：
  * 使用状态机信号控制常驻静态线程池配合双 `pthread_barrier`
 进行多阶段同步，消除频繁创建和锁唤醒开销。
  * 采用 `alignas(64)`
 显式强迫每个线程的局部私有堆独占 CPU 缓存行（Cache Line），彻底阻断伪共享与锁竞争。
  * *局限性*
：由于倒排桶长天然呈偏态分布，静态等长切片会导致严重的负载不均衡，引发长尾空转损耗。
* **Pthread 动态并行**
：
  * 构建**一维展平的原子无锁任务块**，将选中的倒排桶按固定步长（`IVF_TASK_BLOCK`
）切分放入 TaskBlock 队列。
  * 主子线程协同参与计算，通过原子变量 `std::atomic::fetch_add`
 动态抢占任务块，完美抹平偏态长尾空转。
* **OpenMP 并行优化**
：
  * 粗排阶段：采用 `#pragma omp parallel for schedule(static)`
 绑定线程，最大化发挥 L1 Cache 的空间局部性。
  * 精排阶段：采用 `#pragma omp parallel for schedule(dynamic, 1)`
，以单桶为粒度由运行时动态派发，自适应负载均衡。

### 4. IVF 与 PQ 混合索引结合
* **结合范式**：支持 **全局乘积量化 (Global-PQ)** 与 **残差乘积量化 (Residual-PQ)**
 两种拓扑结构。
* **多线程加速**
：分别编写了 Pthread 线程池版本与 OpenMP 版本的混合索引并行检索器，在大幅度裁剪扫描向量数（通过 IVF 倒排）的同时，利用 PQ 低宽带占用的非连续访存特性，进一步压缩查询耗时。

### 5. HNSW 图索引多策略并行优化
针对非连续路由的图索引，本项目实现了三个维度的多线程探索：
* **边划分并行**
：多线程并发切分当前顶点的邻居邻接表，加速多维相似度度量，适用于高维稠密长向量。
* **Layer 0 点划分并行**
：在元素数量最稠密的底层（Layer 0）进行大范围全量切片分片，多核联合搜索目标局部邻域。
* **多入口点并行**
：通过产生随机散射的多入口点，允许多个线程在图的不同拓扑局部并行展开路由，提高探索概率。
